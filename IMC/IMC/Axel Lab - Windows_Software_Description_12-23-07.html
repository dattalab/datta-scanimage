A custom Windows or Matlab application will be created to provide the functionality described below. (The following description assumes network connectivity and network file sharing where network storage / retrieval is indicated):

  • The ability to enter a table of odors and the corresponding bank / port on the Olfactometer.

    • This table will be stored as a CSV format flat file.

    • The application will permit entry & editing of the table.

    • The application will permit storage and retrieval of the table from local or network attached drives.

    • The application will permit designation of a ‘blank’ odor channel for each bank.

  • The ability to enter a table of flow rates for each mass flow controller.

    • The table will be stored as a CSV format flat file.

    • The application will permit entry & editing of the table.

    • The application will permit storage and retrieval of the table from a local or network attached drives.

  • The ability to enter a sequence table corresponding to odor assignments and duration in the Olfactometer.

    • The table will be stored as a CSV format flat file.

    • The application will permit entry & editing of the table.

    • The application will permit storage and retrieval of the table from local or network attached drives.

    • The table will be depicted as a list of odors and delivery duration in sequential format.

    • A pointer will be graphically depicted, such that the current command (the command already sent to the Olfactometer) is identified.

    • The table will depict a short history of the commands to be sent (some number of commands already sent to the Olfactometer, the current command, as well as the commands to be sent.

    • The duration can be programmed to deliver an odor for some number of milliseconds, or continuous (0=continuous).

    • Table entries with non-zero programmed durations will automatically revert to the assigned blank channel for its’ associated bank upon completion of the timed event.

  • The ability to generate commands to the Olfactometer.

    • The application will utilize TCP/IP communication protocol between the application and the Olfactometer.

  • Interfacing with a DIO (Digital I/O board) to accept and deliver a TTL level trigger.

    • The TTL level trigger shall increment the aforementioned pointer.

    • A TTL level ‘completion’ signal shall be available to indicate the execution of the programmed command.

    • The duration of the TTL ‘completion’ signal duration shall be programmable.

  • A graphical method to increment, decrement or reset the pointer to the beginning of the list.

  • The program shall provide the ability to store all of the programmed settings as a project.

    • The stored project shall include embedded copies of all tables.

    • The stored project filename shall be user definable.

    • It shall be possible to store the project on local or network drives.
